## Our publishing rule

We write about the purpose of the work and the evidence we can support. We do
not publish the private implementation, a product-sensitive method, or an
unfinished research idea simply to make the project sound more impressive.

When a capability changes, the [progress record](EVIDENCE.md) should change
before stronger language appears elsewhere.
