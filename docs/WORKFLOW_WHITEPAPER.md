# Working with early collaborators

**Updated:** 29 June 2026

**Purpose:** Public evaluation plan

Qalculus Labs is not looking for a vague "innovation pilot." A useful early
evaluation should begin with one real problem, a clear boundary, and an honest
way to decide whether the work helped.

## 1. Start with the problem

The first conversation should identify a specific analytical task that is hard
to run, repeat, govern, or review today. We also need to understand who uses the
result and what a better process would change for them.

If the problem cannot be stated clearly, it is too early to discuss an
integration.

## 2. Agree on the boundary

Before any data or system access is considered, both sides should agree on:

- the narrow question being evaluated;
- the environment in which the work may run;
- which data may be used;
- who may see the outcome;
- what must be logged, retained, or deleted; and
- what would stop the evaluation.

The default starting point is synthetic or safely approved data.

## 3. Define success before building

A pilot should have a small set of meaningful measures. Depending on the task,
these may include correctness, repeatability, review time, operational effort,
cost, or reliability.

The existing process should be the baseline. Without one, it is easy to mistake
a polished demonstration for an improvement.

## 4. Run a bounded evaluation

The first evaluation should test one supported path. It should not attempt to
prove that the platform handles every method, data source, or deployment model.

During the evaluation, we will record what worked, what failed, what remained
manual, and which assumptions affected the outcome.

## 5. Review the result together

At the end, the decision should be straightforward:

- continue because the evidence supports a larger next step;
- revise the scope and test again; or
- stop because the product did not create enough value in that setting.

A decision to stop can still be a successful evaluation if it was reached
quickly and honestly.

## Current availability

This is a future collaboration model, not an announcement of an open pilot
programme. Qalculus Labs is currently strengthening the product foundation and
its security and evaluation practices.
